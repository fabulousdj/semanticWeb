Semantic Modeling


Current Models Stink (at talking)

  • It's all about the way you model your data. Some common models you are probably familiar with are:
    • Relational: SQL
    • Hierarchical: XML
  • But these don't play well together... even within the same model. If you want to combine and compare data between two SQL or XML sources, you need to know highly detailed specifics about their structure. This takes time, money and coordination between you and the writers of the data stores.
    • Decide on a common data format
    • Create a secure endpoint on each website for requests
    • Want to incorporate a third source? Start all over again.

Why Semantic Modeling is Better

  • Semantic modeling makes it easy to communicate between data sources. Here are some important terms you should know before we dive in:
    • Vocabulary: A collection of terms given a well-defined meaning that is consistent across contexts.
    • Ontology: Allows you to define contextual relationships behind a defined vocabulary. See OWL (Web Ontology Language) for a formal example. More on this later.
  • Now let's consider an example of two sites with similar information that we may want to query from. Let's say:
    • Site 1: A recipe database. Lots of info on ingredients, cooking times, etc.
    • Site 2: A Food Network site that keeps track of meals that have been cooked on their shows, along with variations of the recipes.
  • You are building a hands-free kitchen helper that pulls recipe info, and suddenly your user asks, "What's an easy way to cook the last thing Rachel Ray made on Food Network?" How do you write that query?
    • With the semantic web, this is easy!
    • If Site 1 and Site 2 adopt a common vocabulary such that they both agree on what a "recipe", "ingredient", and "chef" is, you can query both sites by using exactly the same terms.

Not All Fun and Games

  • Of course, the "if" in the phrase "if both sites adopt a common vocabulary" is a BIG "if".
    • There are common vocabularies out there that anyone can use (remember OWL)
    • But adding it to your site means overhead.
  • Maybe one day the whole internet will be semantic-enabled. But for now, most initiatives to become semantic are within smaller knowledge domains:
    • Dublin Core Metadata Initiative (DCMI) - Creates ontologies for a range of subjects, particularly focusing on common, every day terms and terms important in media.
    • Friend Of A Friend (FOAF) - focuses on developing a standard vocabulary/ontology for social networking purposes.
    • OpenCyc - An ontology of everyday, common sense terms.
